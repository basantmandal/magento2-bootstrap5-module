## Reporting a Vulnerability

We take the security of **HK2 AddBootstrap5** seriously. If you discover a security vulnerability, please report it responsibly.

**How to report:**

1. **DO NOT** create a public GitHub issue for security vulnerabilities
2. Email us at **[support@basantmandal.in](mailto:support@basantmandal.in)**
3. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce (with proof of concept if possible)
   - Potential impact assessment
   - Suggested fix (if any)

**Expected response timeline:**

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 5 business days
- **Patch development:** Depending on severity (1-14 days)
- **Public disclosure:** After patch is released

---

## What to Expect

**Our process:**

1. **Confirmation** — We will acknowledge receipt of your report within 48 hours
2. **Investigation** — Our team will investigate and validate the reported vulnerability
3. **Patch Development** — We will develop and test a fix
4. **Release** — A security patch will be released in a new version
5. **Acknowledgment** — With your permission, we will credit you in the release notes

**Confidentiality:**

- We will keep your report confidential until the patch is released
- We will not disclose your identity without your explicit permission
- Please keep the vulnerability confidential until we release the patch

## Scope

**What is covered:**

- The HK2 AddBootstrap5 module codebase
- CDN asset loading mechanisms
- Content Security Policy (CSP) configurations
- Admin configuration interface

**What is NOT covered:**

- Third-party CDN providers (jsDelivr, cdnjs, unpkg)
- Magento 2 core vulnerabilities
- Server infrastructure issues
- Social engineering attacks

---

## Security Best Practices for Users

**Environment Configuration:**

- Never commit `.env` files with sensitive credentials to version control
- Use strong admin passwords for your Magento store
- Enable two-factor authentication (2FA) for admin access

**Network Security:**

- Restrict admin panel access by IP when possible
- Use HTTPS for all storefront and admin traffic
- Keep your server firewall rules updated

**Regular Updates:**

- Keep the module updated to the latest version
- Keep Magento 2 and all dependencies up to date
- Monitor the changelog for security patches
